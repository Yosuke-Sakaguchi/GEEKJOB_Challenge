1.Spring Framework
概要
公式サイトでは「スプリングフレームワークは近代的なJavaの業務アプリ開発の包括的なプログラミング環境を供給します。」と書かれています。要はスプリングはカバーしている範囲がとても広い万能型フレームワークだということです。日本でも世界でもかなり広く使われているものであり、また、万能なフレームワークなので様々なシステム構築に使用可能なものです。

導入方法
Spring Frameworkを利用するための統合開発環境である「Spring Tool Suite」を利用する

2.Play Framework
概要
公式サイトでは「Playは、現代のwebアプリケーション開発に必要なコンポーネント及びAPIを統合した生産性の高いJavaとScalaのwebアプリケーションのフレームワークです。Playの特徴は、ライトウェイト、ステートレス、webフレンドリーなアーキテクチャであること、機能予測のしやすさです。また、IterateeIOをベースにしたリアクティブモデルのおかげで、スケーラブルなアプリケーションでもCPU、メモリ、スレッドなどのリソース消費が最小限になっています。」と書かれています。JavaとScalaをサポートしているフレームワークで、使いやすくて開発しやすいことを重視したフレームワークです。

導入方法
参考サイト→http://mofmof721.hatenablog.com/entry/2015/12/27/002034
JDK、PlayFrameworkをインストールする

3.JSF(JavaServerFaces)
概要
公式サイトでは「JSFはJavaEEという環境で動作する標準的なコンポーネント志向(機能ごとに部品として扱う考え方)のユーザーインターフェイスフレームワークです。JSFはJavaEEに含まれているので、新しくライブラリなどをプロジェクトに追加しなくても使うことができます。」と書かれています。要はUIを簡単に作ることができるフレームワークだということです。
スタンダードに使われている技術であり、新しい技術ではないので綺麗な画面を作るということはあまり得意ではないが、実際利用している企業が多いので仕事としては利用することがあります。

導入方法
参考サイト→http://qiita.com/yu_naka0607/items/c3441175a90be72c2b6f
JDK,Eclipse JBoss EAP,Jbossツールをインストール
